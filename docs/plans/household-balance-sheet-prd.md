# 家庭资产负债表 PRD（V1）

- 产品形态：本地桌面应用优先（Electron 封装，本机运行，无登录）
- 版本范围：V1（单家庭）

## 1. 背景与目标
用户需要在本地管理家庭资产与负债，核心诉求是：可持续录入、可按成员归属、可按固定分类管理、可形成时间序列分析，并通过直观图表发现配置偏离与再平衡机会。V1 目标如下：
1. 建立稳定的数据录入与导入能力（手动 + CSV）。
2. 支持多币种金额录入，自动获取当日汇率并统一折算。
3. 建立自动快照与日频序列，支持趋势、波动率、相关性分析。
4. 提供家庭资产负债桑基图与再平衡提醒，帮助识别超配/低配。
5. 以桌面安装包形式交付给非技术用户，保证可感知的启动反馈与一键进入使用。

## 2. 产品范围
### 2.1 In Scope（V1）
1. 单家庭模型（仅 1 个家庭）。
2. 家庭成员管理（独立菜单，可新增/删除成员）。
3. 资产与负债录入（每笔必须归属成员）。
4. 固定三级分类结构（资产/负债分别三级；层级深度固定，不可自定义）。
5. 多币种录入 + 自动拉取当日汇率折算到基准币。
6. CSV 导入（更新覆盖 + 新增）。
7. 自动快照（每次新增/编辑触发）+ 每日日终快照。
8. 可视化：趋势图、桑基图、波动率、相关性矩阵、再平衡提醒。
9. 系统设置：基准币种、时区（默认本机时区且只读）、汇率提供方（固定 frankfurter）、全局偏离阈值（用户自定义百分比）。
10. 桌面版启动壳：应用图标、加载态、失败页与本地 sidecar 启动流程。

### 2.2 Out of Scope（V1 不做）
1. 多家庭支持。
2. 账号体系与权限分级。
3. 自定义层级深度。
4. 自动对接券商/银行账户。
5. IRR/XIRR 计算与展示。

## 3. 用户与使用场景
### 3.1 目标用户
1. 家庭资产负债表维护者（本机使用者）。

### 3.2 关键场景
1. 日常手动录入或编辑资产/负债。
2. 周期性通过 CSV 批量导入更新。
3. 查看净资产趋势与结构变化。
4. 查看资产波动率和资产相关性，识别风险集中。
5. 按目标占比检查偏离并执行再平衡。

## 4. 信息架构与页面
一级导航：`总览`、`分析看板`、`资产负债录入`、`成员管理`、`CSV导入`、`设置`。

1. 总览
- 展示：总资产、总负债、净资产、最近快照时间、异常提示。

2. 资产负债录入
- 表格 + 表单抽屉。
- 支持新增、编辑、删除（可选软删除）。
- 保存后自动生成事件快照。
- 成员仅用于归属选择，成员维护在独立菜单页。

3. 成员管理
- 独立页面管理家庭成员。
- 支持成员新增与删除。
- 删除失败时返回明确业务错误（例如成员已被持仓占用）。

4. CSV导入
- 三步：上传 -> 字段映射 -> 预检确认。
- 展示将更新/新增/失败条数。
- 导入完成生成批次日志与错误明细。

5. 分析看板
- 页面内提供二级分析视角切换：`整体概览`、`风险与配置`、`币种总览`。
- `整体概览` 与 `风险与配置` 共享时间区间筛选器，筛选器应采用整块可点击的卡片式交互，而不是仅依赖小图标触发。
- `整体概览`：净资产趋势（时间序列）、家庭资产负债桑基流向图。
- `风险与配置`：资产波动率排名、资产相关性矩阵、再平衡提醒列表（超配/低配）。
- `币种总览`：按币种查看资产、负债、净资产、各项资产/负债占比与明细表。

6. 设置
- 基准币种：下拉选择常见币种（如 `CNY 人民币`、`USD 美元`）。
- 时区：默认读取本机时区，字段只读不可编辑。
- 汇率提供方：固定为 `frankfurter`，只读不可编辑。
- 全局偏离阈值：自定义百分比。

## 5. 关键功能需求
### 5.1 成员管理
1. 默认存在 1 个家庭。
2. 成员管理为独立菜单页面。
3. 可新增、删除成员名称。
4. 删除存在关联持仓的成员时，返回业务错误并提示先处理持仓。
5. 每笔资产/负债必须绑定 1 位成员。

### 5.2 资产/负债录入
1. 字段（必填）：名称、类型（资产/负债）、成员、三级分类、币种、金额。
2. 资产额外字段（必填）：期望占比（%）。
3. 金额模型：仅总金额，不支持数量/单价。
4. 分类模型：固定三级，V1 内置面向中国家庭用户的精简分类，前后端默认使用同一套分类树。

#### 5.2.3 分类使用规则
1. `成员` 已单独建模，分类中不再重复表达家庭成员身份。
2. 机构名或账户名应填写在 `名称` 字段，例如“招商银行活期”“沪深300ETF”“杭州自住房”。
3. V1 不支持用户自定义分类层级或在线维护分类树；如需扩展，后续通过种子数据或迁移脚本升级。

### 5.3 汇率与折算
1. 每日自动获取当日汇率。
2. 多币种金额实时折算为基准币用于统计。
3. 当日汇率缺失时，回退最近可用日期并标记估算状态。
4. V1 汇率提供方固定为 `frankfurter`，设置页不提供切换能力。

### 5.4 CSV 导入
1. 支持模板下载与字段映射。
2. 覆盖键：`名称 + 成员 + 三级分类路径`。
3. 命中覆盖键则更新；未命中则新增。
4. 导入需支持预检和错误报告下载。

### 5.5 快照
1. 事件快照：每次新增/编辑后自动写入。
2. 日终快照：每日生成 1 条日频快照（用于统计）。

### 5.6 分析与提醒
1. 趋势图：净资产、总资产、总负债随日期变化。
2. 波动率：基于日频收益率序列计算并展示年化波动率。
3. 相关性矩阵：基于日频收益率计算 Pearson 相关系数。
4. 桑基图：完整展示家庭资产与负债流向结构。
5. 再平衡提醒：基于资产目标占比与当前占比偏离触发。

## 6. 计算口径
1. 总资产 = 所有资产项折算金额之和。
2. 总负债 = 所有负债项折算金额之和。
3. 净资产 = 总资产 - 总负债。
4. 资产当前占比 = 资产项折算金额 / 净资产。
5. 偏离度 = 当前占比 - 期望占比。
6. 触发条件：`|偏离度| >= 全局阈值`。
7. 再平衡标签：
- 偏离度 > 0：超配。
- 偏离度 < 0：低配。

边界规则：
1. 当净资产 <= 0 时，占比、偏离度、再平衡提醒不计算，页面显示“净资产非正，指标不可用”。
2. 相关性与波动率样本不足（默认 < 30 个日频点）时，显示“数据不足”。
3. 相关性矩阵中后端返回 `None` 的单元格，前端显示 `N/A / 样本不足`，不得当作 `0` 相关处理。

3. `category`

## 8. 非功能需求
1. 部署与运行
- 本地一键启动（桌面壳 + 本地后端 + 内嵌前端）。
- 默认仅 localhost 访问。
 - 首次启动应提供明确的加载反馈，避免用户误以为应用无响应。

2. 性能
- 单次列表加载（1,000 条以内）响应 < 1s（本地环境目标）。
- CSV 导入 5,000 行内可完成预检与写入。
 - 桌面版冷启动应尽量控制在可接受范围内，避免超过用户耐心阈值，并用加载态覆盖启动等待。

3. 可靠性
- 关键写操作事务化。
- 导入失败不影响已存在数据一致性。

4. 可维护性
- 所有计算逻辑集中在服务层。
- 导入、汇率、快照任务具备日志。

## 9. 异常与错误处理
1. 汇率源不可用：回退最近可用汇率，显示黄色状态提示。
2. CSV 字段缺失：阻断导入并返回列级错误。
3. CSV 行数据非法：记录行号与原因，允许部分成功。
4. 快照生成失败：阻断“已成功保存”提示并记录错误日志。
5. 应用错误按错误码映射 HTTP 状态：`4040 -> 404`、`4090 -> 409`、`5000 -> 500`、其他业务错误默认 `400`。

## 10. 验收标准（V1）
1. 可通过桌面应用在本机无登录访问并完成基础操作。
2. 可新增成员并把资产/负债正确归属到成员。
3. 可完成手动录入与 CSV 导入，且覆盖键行为正确。
4. 可自动获取汇率并完成多币种折算。
5. 每次新增/编辑后可见事件快照；每日可生成日终快照。
6. 可正确展示趋势图、桑基图、波动率、相关性矩阵。
7. 设置页中汇率提供方固定为 `frankfurter` 且不可编辑；时区默认本机且不可编辑；基准币种为下拉选项。
8. 相关性矩阵的样本不足单元格显示 `N/A / 样本不足`，不与零相关混淆。
9. 左侧导航固定在顶部，顺序为 `总览 -> 分析看板 -> 资产负债录入 -> 成员管理 -> CSV导入 -> 设置`，且包含独立“成员管理”入口。
10. 分析看板时间区间组件支持整块点击触发日期选择，交互面积足够大。
11. 当净资产 <= 0 或样本不足时，系统按规则给出清晰提示。

## 11. 里程碑建议
1. M1（数据底座）
- 完成实体、录入、成员、内置三级分类、基础列表。

2. M2（导入与汇率）
- 完成 CSV 预检/导入、汇率拉取与折算。

3. M3（快照与分析）
- 完成事件快照、日终快照、趋势/波动率/相关性。

4. M4（可视化与提醒）
- 完成桑基图、再平衡提醒、设置页和验收回归。

## 12. 风险与约束
1. 汇率源稳定性影响当日折算精度。
2. 分类树一旦进入真实使用，需要审慎维护命名与排序，避免影响 CSV 导入、历史持仓和分析口径。
3. 本地无登录模式安全边界依赖设备环境，不适用于多用户共享设备。
